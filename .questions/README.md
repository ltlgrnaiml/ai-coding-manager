# Questions Directory

Open questions requiring human input.

## Purpose

When an AI assistant encounters ambiguity or needs clarification, it should create a question file here rather than guessing.

## Question Naming

```
Q-XXX_<brief_topic>.md
```

Example: `Q-001_Database-Choice.md`

## When to Create Questions

- Requirements conflict
- Architecture decisions need human input
- Multiple valid approaches exist
- Unclear acceptance criteria

## Question Template

```markdown
# Q-XXX: Brief Topic

**Date**: YYYY-MM-DD
**Context**: What work prompted this question
**Status**: Open | Resolved

## Question

Clear statement of what needs clarification.

## Options Considered

1. **Option A**: Description
   - Pros: ...
   - Cons: ...

2. **Option B**: Description
   - Pros: ...
   - Cons: ...

## Recommendation

AI's recommended approach (if any).

## Resolution

[Filled in when resolved]
- **Decision**: What was decided
- **Rationale**: Why
- **Resolved by**: Who
- **Date**: When
```
